# Cafe Management System

Веб-приложение для автоматизации работы небольшого кафе, разработанное на Django.

## Функциональность

### Управление столами
- Создание, редактирование, удаление столов
- Управление количеством мест
- Активация/деактивация столов

### Управление меню
- Добавление и редактирование блюд
- Установка цен
- Управление доступностью блюд

### Управление заказами
- Создание заказов
- Добавление блюд в заказ
- Закрытие и оплата заказов
- История заказов

### Управление сменами
- Открытие и закрытие смен
- Подсчёт выручки за смену
- История смен

### Роли пользователей
- **Администратор**: полный доступ ко всем функциям
- **Официант**: работа со столами и заказами

## Технологический стек

- **Backend**: Python 3.12+, Django 6.0
- **Database**: SQLite (для разработки)
- **Frontend**: Django Templates, HTML5, CSS3, Bootstrap 5
- **Тестирование**: Django TestCase (15 тестов, все проходят успешно ✅)

## Быстрый старт

### 1. Установка

```bash
# Перейдите в директорию проекта
cd cafe_management

# Создайте виртуальное окружение
python -m venv venv

# Активируйте виртуальное окружение
.\venv\Scripts\activate  # Windows

# Установите зависимости
pip install -r requirements.txt
```

### 2. Настройка базы данных

```bash
# Примените миграции
python manage.py makemigrations
python manage.py migrate

# Создайте суперпользователя (или используйте populate_db)
python manage.py createsuperuser
```

### 3. Заполнение тестовыми данными (опционально)

```bash
python manage.py populate_db
```

Это создаст:
- Пользователя `admin` с паролем `admin123`
- Пользователя `waiter` с паролем `waiter123`
- 5 столов
- 8 блюд из меню
- Открытую смену

### 4. Запуск сервера

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: http://127.0.0.1:8000/

## Модели базы данных

### Table (Стол)
- `number` - Номер стола (уникальный)
- `seats` - Количество мест
- `is_active` - Статус активности (soft delete)

### Dish (Блюдо)
- `name` - Название
- `description` - Описание
- `price` - Цена (> 0, валидация)
- `is_available` - Доступность

### Shift (Смена)
- `opened_at` - Время открытия (авто)
- `closed_at` - Время закрытия
- `is_open` - Статус открытия

### Order (Заказ)
- `table` - Ссылка на стол (PROTECT)
- `shift` - Ссылка на смену (PROTECT)
- `status` - Статус (open/paid/cancelled)
- `created_at` - Время создания (авто)
- `total_price` - Вычисляемое свойство

### OrderItem (Позиция заказа)
- `order` - Ссылка на заказ (CASCADE)
- `dish` - Ссылка на блюдо (PROTECT)
- `quantity` - Количество
- `subtotal` - Вычисляемое свойство

## Бизнес-правила

1. ✅ Номер стола уникален
2. ✅ Одновременно может существовать только одна открытая смена
3. ✅ Заказ можно создать только при открытой смене
4. ✅ Оплаченный заказ нельзя редактировать
5. ✅ Недоступные блюда нельзя добавлять в заказ (валидация в форме)
6. ✅ Цена блюда должна быть больше 0
7. ✅ Нельзя создать два открытых заказа за одним столом

## Разработка

### Добавление нового функционала

1. Создайте модель в `models.py`
2. Создайте миграции: `python manage.py makemigrations`
3. Примените миграции: `python manage.py migrate`
4. Зарегистрируйте модель в `admin.py`
5. Создайте views и формы
6. Создайте шаблоны в `templates/`
7. Добавьте URL-маршруты
8. Напишите тесты
